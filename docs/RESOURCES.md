# AEGIS — Resources, Datasets, Standards & References

Everything AEGIS is built on, with links. Grouped by category.

## Datasets

**CMU CERT Insider Threat Test Dataset (r4.2)** — the standard synthetic
benchmark for insider-threat research, generated by CMU's Software Engineering
Institute / CERT with ExactData under DARPA I2O. Release r4.2 contains ~32.8M
events from 1,000 users over Jan-2010→May-2011 with 7,323 labeled malicious
instances across logon, device, file, http, and email logs plus LDAP org data
and an `answers/` folder of ground-truth malicious actors. AEGIS ships a loader
(`data/cert_loader.py`) that maps it onto the platform's event model.

- Dataset home: https://www.sei.cmu.edu/library/insider-threat-test-dataset/
- Figshare download: https://kilthub.cmu.edu/articles/dataset/Insider_Threat_Test_Dataset/12841247

For an offline demo, AEGIS's own deterministic simulator (`data/generator.py`)
generates a realistic bank workforce and injectable attack scenarios — no
download required.

## Post-Quantum Cryptography standards

NIST finalized the first post-quantum standards on 13 Aug 2024. AEGIS targets
the NIST security-category-3 parameter sets.

- **FIPS 203 — ML-KEM** (Module-Lattice Key Encapsulation, from CRYSTALS-Kyber): https://csrc.nist.gov/pubs/fips/203/final
- **FIPS 204 — ML-DSA** (Module-Lattice Digital Signature, from CRYSTALS-Dilithium): https://csrc.nist.gov/pubs/fips/204/final
- **FIPS 205 — SLH-DSA** (stateless hash-based signatures, from SPHINCS+): https://csrc.nist.gov/pubs/fips/205/final
- NIST PQC standardization overview: https://en.wikipedia.org/wiki/NIST_Post-Quantum_Cryptography_Standardization

Context on migration timelines (NSA CNSA 2.0 targets 2030 for national-security
systems) motivates the "harvest-now, decrypt-later" threat AEGIS defends
against.

## Cryptography libraries

- **liboqs / liboqs-python** (Open Quantum Safe) — production PQC, AEGIS tier 1: https://pypi.org/project/liboqs-python/ · https://openquantumsafe.org/
- **kyber-py** — pure-python ML-KEM (FIPS 203), AEGIS tier 2 KEM: https://pypi.org/project/kyber-py/ · https://github.com/GiacomoPope/kyber-py
- **dilithium-py** — pure-python ML-DSA (FIPS 204), AEGIS tier 2 signatures: https://pypi.org/project/dilithium-py/ · https://github.com/GiacomoPope/dilithium-py
- **cryptography** (pyca) — AES-256-GCM, HKDF, X25519/Ed25519 for the vault, hybrid mode, and classical fallback: https://cryptography.io/

## Threat frameworks & regulatory guidance

- **MITRE ATT&CK for Enterprise** — the technique taxonomy AEGIS rules map to: https://attack.mitre.org/
- **Center for Threat-Informed Defense — Insider Threat TTP Knowledge Base**: https://center-for-threat-informed-defense.github.io/insider-threat-ttp-kb/
- **Applying MITRE ATT&CK to insider threats** (Securonix): https://www.securonix.com/blog/applying-the-mitre-attck-framework-to-insider-threats/
- **CISA — compromised former-employee account advisory (AA24-046A)**: https://www.cisa.gov/news-events/cybersecurity-advisories/aa24-046a
- **RBI Cyber Security Framework for Banks** — mandates PAM with session recording, least-privilege, maker-checker, periodic access reviews, and monitoring of privileged activity. Reference guides: https://www.upguard.com/blog/rbi-cybersecurity-framework-baseline-requirements · https://arconnet.com/the-rbi-guidelines-on-cybersecurity-framework-in-banks/
